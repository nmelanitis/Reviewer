Comparison of REGEX and LLM classifications
===========================================

Input file:
MAEllm/classified_papers_with_review_flags.csv

The file compares two classification runs:

1. MAEllm/classified_papers.csv
   - Direct LLM classification.
   - Uses extracted full text when available.
   - Falls back to abstracts when full text is missing.

2. MAEex/classified_papers.csv
   - Earlier REGEX-based classification.
   - Uses generated regular expressions over title + abstract.


Meaning of not_yes_in_maeex
---------------------------

not_yes_in_maeex=no
means:

    The paper WAS classified as "yes" in MAEex/classified_papers.csv.

Equivalently:

    maeex_verdict == yes


not_yes_in_maeex=yes
means:

    The paper was NOT classified as "yes" in MAEex/classified_papers.csv.

So the MAEex verdict was one of:

    no
    dont_know
    missing/unmatched

In the current comparison, all 195 papers matched between the two CSV files, so
there were no missing/unmatched MAEex rows.
